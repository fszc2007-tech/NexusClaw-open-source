# Demo Project

This folder contains small, fictional sample documents for testing NexusClaw locally.

Use these files to validate the first-run flow:

1. Start the backend and frontends.
2. Create or select a project in the admin console.
3. Create a knowledge base.
4. Upload one or more files from `examples/demo-project/docs`.
5. Ask questions in the portal.
6. Inspect the answer and source context.

## Sample Questions

- What documents are required for remote work approval?
- How long does expense reimbursement usually take?
- What is the difference between a deductible and coinsurance?
- What should a resident prepare before booking a service appointment?
- How should urgent facility maintenance be reported?

## Expected Behavior

The assistant should answer from the uploaded documents, keep the answer concise, and show source-aware context where the UI/API exposes it.

These files are fictional examples. They are not legal, medical, financial, HR, or government advice.
